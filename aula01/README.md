# Aula 1 — Sistema de Empréstimo de Equipamentos (v1.0)

Código legado desta aula. **Funciona, mas é estruturalmente frágil** — é o que você vai diagnosticar.

## Como rodar

No terminal do **GitHub Codespaces** (tudo roda no navegador, sem instalar nada):

```bash
cd aula01
python emprestimos.py
```

## Conteúdo

- `emprestimos.py` — o sistema (v1.0).
- `docs/` — visão, requisitos, casos de uso e projeto (com a tabela de dívida técnica).

## Ordem de leitura sugerida

Leia do mais amplo ao mais próximo do código e, por último, o código — comparando tudo com ele:

1. **`docs/visao.md`** — o quê e para quem (propósito, escopo, stakeholders).
2. **`docs/requisitos.md`** — o que o sistema deve fazer (RF, regras de negócio, RNF, requisitos de interface).
3. **`docs/casos_de_uso.md`** — como se usa (fluxos principais e alternativos).
4. **`docs/projeto.md`** — como foi construído (diagrama de classes + **tabela de dívida técnica**).
5. **`emprestimos.py`** — por último, lendo o código e **comparando** com 1–4: é aí que as divergências aparecem.
